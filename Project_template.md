### 1. Описание функциональности монолитного приложения

**Управление отоплением:**

- Пользователи могут удаленно включать и выключать отопление в своих домах
- Пользователи могут устанавливать желаемую температуру в доме
- Система поддерживает удаленный запуск и остановку работы отопительных установок
- Система поддерживает установку желаемой температуры в доме
- Система поддерживает установку, замену и обновление отопительных установок

**Мониторинг температуры:**

- Пользователи могут просматривать текущую температуру в своих домах через веб-интерфейс
- Система поддерживает подключение датчиков температуры
- Система поддерживает считывание показаний текущей температуры с датчиков и передачу информации на сервер

### **4. Проблемы монолитного решения**

- Слабая масштабируемость системы 
  - При росте числа пользователей увеличивается нагрузка на единую базу данных.\
  - При необходимости масштабировать любой отдельно взятый домен приходится масштабировать все приложение
- Низкая отказоустойчивость
  - Отказ любого компонента приложения может вести к отказу всего приложения
  - Единая база данных является слабым местом. В случае отказа все приложение становится недоступным
  - Долгая перезагрузка приложения. В случае отказа приложения приходится перезапускать весь сервис
- Проблемы при обновлении приложения
  - При обновлении любого из компонентов приходится перезапускать все приложение
  - При развертывании новой фичи есть риск сломать все приложение. Следовательно, увеличивается сложность и время тестирования
- Сложность интеграций с новыми устройствами
  - При добавлении нового типа устройства приходится вносить изменения во всю кодовую базу. Как следствие, приходится проводить тестирование всего приложения
  - Если устройства используют разные API или протоколы, то приходится адаптировать все приложение, что усложняет архитектуру
- Проблемы при разработке
  - На проекте работает несколько разработчиков. Изменения в коде одним разработчиком могут влиять на работу остальных
  - Долгий и сложный онбординг новых сотрудников - им приходится изучать работу всего приложения
  - Затянутые релизы. При разработке новых фич приходится тестировать всю систему, что замедляет процесс релизов
